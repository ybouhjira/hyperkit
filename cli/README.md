# HyperKit CLI

Command-line tool for HyperKit theme generation and utilities.

## Installation

When you install `@ybouhjira/hyperkit`, the CLI is automatically available:

```bash
npm install @ybouhjira/hyperkit
```

## Commands

### `hyperkit init`

Initialize a HyperKit theme in your project.

```bash
hyperkit init [--dir <directory>]
```

**Options:**

- `--dir, -d <directory>` - Target directory (default: current directory)

**What it does:**

- Creates a `theme.ts` file with a complete ThemeConfig template
- Provides usage instructions for integrating with ThemeProvider

**Example:**

```bash
hyperkit init
hyperkit init --dir ./src
```

### `hyperkit theme generate`

Generate TypeScript types from your theme configuration.

```bash
hyperkit theme generate [--input <file>] [--output <file>]
```

**Options:**

- `--input, -i <file>` - Theme config file (default: `./theme.ts`)
- `--output, -o <file>` - Output declaration file (default: `./hyperkit-theme.d.ts`)

**What it does:**

- Reads your theme file
- Extracts custom property keys from `customProperties`
- Generates TypeScript declaration file for type-safe CSS variable access

**Example:**

```bash
hyperkit theme generate
hyperkit theme generate --input ./src/theme.ts --output ./types/theme.d.ts
```

**Generated output:**

```typescript
// Auto-generated by hyperkit CLI
// Do not edit manually - run "hyperkit theme generate" to regenerate

declare module '@ybouhjira/hyperkit' {
  interface CustomThemeProperties {
    'header-height': string;
    'sidebar-width': string;
  }
}

export {};
```

### `hyperkit tokens`

List all available design tokens.

```bash
hyperkit tokens
```

**What it does:**

- Displays all HyperKit token types and their allowed values
- Shows token categories: Space, FontSize, FontWeight, Bg, TextColor, Radius, Shadow, Z-Index

**Example output:**


## Development

### Running locally (development mode)

```bash
npm run cli:dev -- <command>
```

Example:

```bash
npm run cli:dev -- init
npm run cli:dev -- tokens
npm run cli:dev -- theme generate
```

### Building the CLI

```bash
npm run cli:build
```

This creates a bundled, production-ready CLI in `cli/dist/index.js`.

## How it Works

The CLI is built with:

- **Commander.js** - Command-line argument parsing
- **esbuild** - Fast bundling for production
- **tsx** - TypeScript execution for development

The CLI is automatically built as part of the main build process (`npm run build`).
